☁️ Cloudflare Integration

Connect your Cloudflare Worker for secure data storage, image hosting, and instant publishing.

⚡
How it works: Your admin dashboard calls a Cloudflare Worker API. The Worker stores product data in Cloudflare KV and serves images via Cloudflare Images CDN. No GitHub tokens needed — all secrets live securely in Cloudflare.
Found in: Cloudflare Dashboard → Workers & Pages → your worker → Settings → Triggers
Found in: Cloudflare Dashboard → Images → Overview → Account Hash

🚀 Setup Guide

Follow these steps to deploy the full Cloudflare stack.

1 Deploy Cloudflare Worker
  1. Install Wrangler CLI: npm install -g wrangler
  2. Login: wrangler login
  3. Create KV namespace: wrangler kv:namespace create PRODUCTS_KV
  4. Copy the returned ID into wrangler.toml → id = "YOUR_ID_HERE"
  5. Set your Cloudflare Account ID in wrangler.toml → CF_ACCOUNT_ID
  6. Set secrets securely (one at a time — they prompt for input):
    wrangler secret put ADMIN_USERNAME
    wrangler secret put ADMIN_PASSWORD
    wrangler secret put JWT_SECRET
    wrangler secret put CF_IMAGES_TOKEN
  7. Deploy: wrangler deploy
  8. Copy the Worker URL (e.g. https://alarouba-products-api.xxx.workers.dev) and paste it above.
2 Enable Cloudflare Images
  1. Go to Cloudflare Dashboard ↗ → Images
  2. Enable Cloudflare Images (free tier: 100,000 images/month)
  3. Create an API token with Cloudflare Images: Edit permission
  4. Run: wrangler secret put CF_IMAGES_TOKEN → paste the token
  5. Copy your Account Hash from the Images overview page
  6. Paste it as the Images Delivery URL above: https://imagedelivery.net/YOUR_HASH
3 Deploy to Cloudflare Pages
  1. Go to Cloudflare Dashboard ↗ → Workers & Pages → Create application → Pages
  2. Connect your GitHub repository
  3. Set build output directory: leave empty (static HTML site)
  4. Deploy — your site will be live at your-project.pages.dev
  5. Optional: add a custom domain in Pages → Custom Domains
💡
Free tier covers everything: Cloudflare Pages (unlimited), Workers (100,000 req/day), KV (100,000 reads/day), Images (100,000 images/month) — all free for a small pharmacy website.

🐙 GitHub Integration

Push product data directly to your GitHub repository. GitHub Pages will auto-serve the updated JSON.

ℹ️
How it works: When you click Publish to GitHub, the dashboard pushes your updated data/products.json to your GitHub repo using the GitHub API. GitHub Pages then automatically serves the updated data — no server needed.
Stored in sessionStorage only — never sent anywhere except GitHub API.  Generate token ↗

How to Set Up

  1. Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Classic ↗
  2. Generate a token with repo scope (full repository access).
  3. Paste the token above and fill in your GitHub username, repo name, and branch.
  4. Make sure data/products.json exists in your repository (upload it from the files provided).
  5. Click Test Connection to verify — then Publish to GitHub whenever you make changes.
  6. GitHub Pages serves the updated JSON automatically within ~30 seconds.
💡
Tip: For better security, use Cloudflare Workers instead — your GitHub token never leaves the browser, and all secrets are stored server-side. See the ☁️ Cloudflare Settings tab.
