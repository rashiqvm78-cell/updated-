GitHub Publishing

Publish your product data directly to GitHub. GitHub Pages will serve the updated JSON automatically within ~30 seconds.

⚠️
Local file detected. You're opening this dashboard directly from your computer (file://). Browsers block all external API calls from local files, which causes the "Failed to fetch" error.

Fix: Serve the dashboard over HTTP — the easiest options are:
• VS Code: Install Live Server extension → right-click dashboard.html → Open with Live Server
• Python: Run python -m http.server 8080 in the project folder → open http://localhost:8080/admin/dashboard.html
• GitHub Pages: Push the whole project to GitHub and open the Pages URL.
🚀
How it works: The dashboard commits your products.json directly to your GitHub repository via the GitHub API. GitHub Pages then serves the updated file to your live website automatically.
Stored in sessionStorage only — never sent anywhere except GitHub API.  Generate token ↗

🚀 How to Set Up

  1. Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Classic ↗
  2. Generate a token with repo scope (full repository access).
  3. Paste the token above and fill in your GitHub username, repo name, and branch.
  4. Make sure data/products.json exists in your repository (upload it from the files provided).
  5. Click Test Connection to verify — then Save Settings.
  6. Use the Publish to GitHub button (or the top-right button) whenever you make changes.
  7. GitHub Pages serves the updated JSON automatically within ~30 seconds.
💡
Tip: Your token is stored in sessionStorage only — it is never saved to disk and will be cleared when you close the tab. Re-enter it each session for security.
